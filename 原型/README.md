# 原型页面使用说明

## 原型入口（建议在 `页面框架.html` 中打开）

- `页面框架.html`（原型首页）

## 文件说明

- `登录页原型.html` - 登录页交互原型

## 功能特性

该原型实现了需求文档中定义的所有功能：

### ✅ 已实现的功能

1. **多种登录方式** - 支持用户名/手机号/邮箱登录
2. **密码显示/隐藏** - 点击眼睛图标切换密码可见性
3. **实时输入验证** - 失去焦点时自动验证格式
4. **验证码功能** - 连续失败3次后自动显示验证码
5. **账号锁定机制** - 连续失败5次锁定30分钟
6. **记住密码** - 勾选后保存账号密码（7天免登录）
7. **错误提示** - 友好的错误提示信息
8. **加载状态** - 登录时显示加载动画
9. **响应式设计** - 支持桌面端、平板端、移动端
10. **回车提交** - 密码框按Enter键提交表单

### 🎨 界面设计

- 配色方案：主色调 #1890ff，错误提示 #ff4d4f，成功提示 #52c41a
- 简洁现代的UI设计
- 流畅的交互动画
- 符合需求文档的设计规范

## 使用方法

### 方式1：直接打开

直接用浏览器打开 `登录页原型.html` 文件即可。

### 方式2：本地服务器（推荐）

```bash
# 使用Python启动本地服务器
python -m http.server 8000

# 或使用Node.js
npx http-server -p 8000
```

然后在浏览器访问：`http://localhost:8000/登录页原型.html`

## 测试场景

### 场景1：正常登录

1. 输入账号：`demo@example.com` 或 `user123` 或 `13800138000`
2. 输入密码：`Test1234`
3. （可选）勾选"记住密码"
4. 点击"登录"按钮
5. **预期结果**：显示"登录成功"提示

### 场景2：密码错误

1. 输入正确账号
2. 输入错误密码（如：`wrong123`）
3. 点击"登录"按钮
4. **预期结果**：显示"用户名或密码错误"提示

### 场景3：触发验证码

1. 连续3次输入错误密码
2. **预期结果**：第3次失败后自动显示验证码输入框
3. 输入验证码（点击图片可刷新）
4. 输入正确的账号密码
5. **预期结果**：可以正常登录

### 场景4：账号锁定

1. 连续5次输入错误密码
2. **预期结果**：显示"账号已被锁定"提示，表单禁用30分钟
3. 等待30分钟后（或清除浏览器localStorage中的`lockUntil`）可重新登录

### 场景5：记住密码

1. 输入账号和密码
2. 勾选"记住密码"
3. 成功登录
4. 刷新页面或重新打开
5. **预期结果**：账号和密码自动填充

### 场景6：输入验证

- 账号为空：显示"请输入用户名/手机号/邮箱"
- 密码为空：显示"请输入密码"
- 密码长度不足8位：显示"密码长度至少8位"
- 密码不包含字母或数字：显示"密码需包含字母和数字"
- 验证码错误：显示"验证码错误"

### 场景7：交互测试

- 点击密码框右侧的眼睛图标：切换密码显示/隐藏
- 点击验证码图片：刷新验证码
- 在密码框按Enter键：提交登录表单
- 账号密码都填写时：登录按钮可用
- 账号或密码为空时：登录按钮禁用（灰色）

### 场景8：响应式测试

- 调整浏览器窗口大小
- 使用移动设备访问
- **预期结果**：布局自动适配，所有功能正常

## 技术说明

### 存储机制

- **localStorage** 用于存储：
  - `savedAccount` - 记住的账号
  - `savedPassword` - 记住的密码（明文，实际应用中应加密）
  - `failedAttempts` - 登录失败次数
  - `lockUntil` - 账号锁定到期时间
  - `token` - 登录Token（演示用）

### 注意事项

⚠️ **这是演示原型，实际应用中需要：**

1. 密码不应以明文存储在localStorage
2. 所有验证应在后端进行
3. 使用HTTPS协议传输数据
4. 实现真正的API接口调用
5. Token应该存储在HttpOnly Cookie中
6. 添加CSRF防护机制
